# API Тестирование Stellar Burgers (Diplom_2)

## Описание
Данный проект содержит автотесты для API сервиса Stellar Burgers. Тесты покрывают основные сценарии работы с пользователями и заказами.

## Структура проекта
```
├── tests/
│   ├── orders/
│   │   ├── test_create_orders.py  # Тесты создания заказов
│   │   ├── test_get_orders.py     # Тесты получения заказов
│   ├── users/
│   │   ├── test_create_users.py   # Тесты создания пользователей
│   │   ├── test_login_users.py    # Тесты логина пользователей
│   │   ├── test_change_users_data.py # Тесты изменения данных пользователей
│
├── requirements.txt  # Зависимости проекта
├── README.md         # Описание проекта
```
## **Технологии**

- **Язык программирования:** Python
- **Библиотеки:** requests, pytest, allure-pytest
- **Стиль тестирования:** Автоматизированное API-тестирование
- **Отчётность:** Allure

## Требования
Перед запуском тестов необходимо установить зависимости:
```sh
pip install -r requirements.txt
```

## Запуск тестов
Тесты можно запускать с помощью `pytest`:
```sh
pytest
```

### Запуск с формированием Allure-отчёта
Чтобы получить отчёт о тестах в Allure, сначала запустите тесты с параметром `--alluredir=allure-results`:
```sh
pytest --alluredir=allure-results
```
Затем сгенерируйте и откройте отчёт:
```sh
allure serve allure-results
```

## Покрываемые сценарии
### **Создание пользователя**
- ✅ Уникальный пользователь
- ✅ Пользователь, который уже зарегистрирован
- ✅ Отсутствие обязательного поля

### **Логин пользователя**
- ✅ Успешный логин
- ✅ Логин с неверными данными

### **Изменение данных пользователя**
- ✅ С авторизацией (изменение `name` и `email`)
- ✅ Без авторизации (ошибка)

### **Создание заказа**
- ✅ С авторизацией
- ✅ Без авторизации
- ✅ С ингредиентами
- ✅ Без ингредиентов (ошибка)
- ✅ С неверным хешем ингредиентов (ошибка)

### **Получение заказов**
- ✅ Авторизованный пользователь
- ✅ Неавторизованный пользователь (ошибка)
